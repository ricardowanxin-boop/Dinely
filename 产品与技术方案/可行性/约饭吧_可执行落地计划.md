### 1.3 解决"无后端多人协作"的关键技术方案

**MVP方案：CloudKit Public Database**

这是实现"无需注册，多人共享约饭局"的关键：

```
CloudKit Public Database（公共库）
├── MealEvent Record（约饭局数据）
│     └── 所有人可读，发起人可写
├── Participant Record（参与人）
│     └── 任何人可添加，只能修改自己的记录
└── Vote Record（投票记录）
      └── 任何人可添加
```

**工作流程：**
1. 发起人创建约饭局，写入CloudKit Public DB，获得记录ID
2. 生成包含该ID的Deep Link（格式：`mealpal://join/[记录ID]`）
3. 好友点击链接，App读取CloudKit记录，展示约饭局
4. 好友填写昵称+Emoji，创建Participant记录，进行投票
5. 所有人实时（或轮询刷新）看到投票结果

**注意：** CloudKit Public DB对未登录用户也可读写，不需要用户有Apple ID登录——这是实现"无需注册"的技术基础。

**Deep Link实现（无需自建服务器）：**
- 注册一个便宜域名（如`mealpal.app`，约$10-15/年），配置`apple-app-site-association`文件
- 或者使用Firebase Dynamic Links（免费）做短链跳转，避免购买域名
- 链接格式：`https://mealpal.app/join/[eventId]`

### 1.4 守约功能设计优化（解决社交压力问题）

**问题：** 仅本地记录守约率，朋友看不到，没有约束力。

**解决方案：公开守约卡片**
- 守约率数据存储在CloudKit中（与参与人记录关联）
- 在约饭局参与人列表中，显示每位参与人的守约率（需本人授权）
- 用户可选择"展示守约率"或"私密"
- 守约率格式：`✓ 张三 守约率 92% (11/12)`

这样，当朋友打开约饭局时，能看到每个人历史守约率，形成真实的社交压力。

### 2.1 技术栈选型（最终确认）

| 技术 | 选型 | 理由 |
|------|------|------|
| 开发语言 | Swift 5.9+ | iOS原生最优 |
| UI框架 | SwiftUI | 比UIKit开发效率高50%，个人开发首选 |
| 云端存储 | CloudKit（Public + Private DB） | 零服务器成本，苹果官方，稳定 |
| 本地存储 | SwiftData（iOS 17+）/ CoreData | 离线数据持久化 |
| 支付 | StoreKit 2 | 原生内购，最简单 |
| 地图POI | 高德地图iOS SDK | 国内最稳定，个人免费额度够用 |
| 多语言 | Localizable.strings | iOS原生国际化 |
| 深度链接 | Universal Links | 需要域名（Firebase Dynamic Links备选） |
| 最低系统要求 | iOS 16+ | 覆盖95%+设备，SwiftUI功能充分 |

**为什么用SwiftUI而非UIKit：**
- 开发效率更高（声明式UI，代码量减少60%）
- 与CloudKit、StoreKit 2的Swift API完美集成
- 原生支持深色模式、动态字体、多语言
- 个人开发者时间成本是最大成本

### 2.4 关键代码实现示例

#### CloudKit写入（创建约饭局）

```swift
import CloudKit

class CloudKitService {
    private let container = CKContainer(identifier: "iCloud.com.yourname.mealpal")
    private let publicDB: CKDatabase
    
    init() {
        publicDB = container.publicCloudDatabase
    }
    
    func createEvent(_ event: MealEvent) async throws -> String {
        let record = CKRecord(recordType: "MealEvent")
        record["eventId"] = event.id
        record["title"] = event.title
        record["creatorName"] = event.creatorName
        record["status"] = "voting"
        record["candidateTimes"] = try JSONEncoder().encode(event.candidateTimes)
        record["maxParticipants"] = 8 // 免费版限制
        
        let savedRecord = try await publicDB.save(record)
        return savedRecord.recordID.recordName
    }
    
    func fetchEvent(by eventId: String) async throws -> MealEvent? {
        let predicate = NSPredicate(format: "eventId == %@", eventId)
        let query = CKQuery(recordType: "MealEvent", predicate: predicate)
        
        let (results, _) = try await publicDB.records(matching: query)
        guard let record = try results.first?.value.get() else { return nil }
        
        return MealEvent(from: record)
    }
}
```

#### StoreKit 2 内购实现

```swift
import StoreKit

class PurchaseService: ObservableObject {
    @Published var isPro: Bool = false
    private let productId = "com.yourname.mealpal.pro"
    
    func purchase() async throws {
        guard let product = try await Product.products(for: [productId]).first else {
            throw PurchaseError.productNotFound
        }
        
        let result = try await product.purchase()
        
        switch result {
        case .success(let verification):
            switch verification {
            case .verified(_):
                isPro = true
                UserDefaults.standard.set(true, forKey: "isPro")
            case .unverified:
                throw PurchaseError.verificationFailed
            }
        case .userCancelled:
            break
        case .pending:
            break
        @unknown default:
            break
        }
    }
    
    func restorePurchases() async throws {
        try await AppStore.sync()
        for await result in Transaction.currentEntitlements {
            if case .verified(let transaction) = result,
               transaction.productID == productId {
                isPro = true
                UserDefaults.standard.set(true, forKey: "isPro")
            }
        }
    }
}
```

### 2.5 Deep Link实现

**方案：Firebase Dynamic Links（免费，无需域名）**

```swift
// AppDelegate.swift
func application(_ app: UIApplication,
                 open url: URL,
                 options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
    if url.scheme == "mealpal" {
        DeepLinkService.shared.handle(url)
        return true
    }
    return false
}

// DeepLinkService.swift
class DeepLinkService {
    static let shared = DeepLinkService()
    
    func handle(_ url: URL) {
        // mealpal://join/[eventId]
        guard url.host == "join",
              let eventId = url.pathComponents.last else { return }
        
        // 发送通知，让首页跳转到对应约饭局
        NotificationCenter.default.post(
            name: .joinEvent,
            object: nil,
            userInfo: ["eventId": eventId]
        )
    }
    
    func generateShareLink(for eventId: String) -> String {
        // 生成分享链接（Firebase Dynamic Links短链）
        return "https://mealpal.page.link/?eventId=\(eventId)"
    }
}
```

## 九、关键风险与应对预案

### 风险1：CloudKit在中国大陆不稳定
**预案：** 大陆版降级为纯本地模式（CloudKit失败时自动降级），约饭局数据通过微信直接发送JSON文本（备用分享方案），保证功能可用性。

### 风险2：冷启动无流量
**预案：** 提前准备好抖音/小红书账号，第一条视频在上线当天发布；同步在V2EX、少数派、AppSo投稿；找5-10个真实朋友发起真实约饭局，截图/录屏作为素材。

### 风险3：App Store审核被拒
**常见拒绝原因及预防：**
- 内购描述不清晰 → 提前写清楚每项权益
- 截图与功能不符 → 所有截图用真实App截图
- Deep Link处理不当 → 提交前测试所有跳转路径

### 风险4：用户转化率低于预期
**预案：** A/B测试内购定价（¥6 vs ¥8 vs ¥12）；优化付费墙设计（明确展示付费前后的差异）；考虑增加"7天免费试用"降低付费门槛。
